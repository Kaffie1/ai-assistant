# Phase C：稳定运营实现文档

## 1. 阶段目标

- 让系统从“可用”进入“可长期运营”
- 提升检索质量稳定性与可观测性
- 建立隐私策略与审计闭环

## 2. 范围边界

包含：
- 记忆治理任务（去重、衰减、压缩、归档）
- 质量评估与指标体系
- 策略中心与审计日志

不包含：
- 模型训练或权重微调平台
- 复杂多租户权限平台（仅预留接口）

## 3. 模块设计（计划）

1. `memory/maintenance.py`（新）
- 周期性治理任务入口
- 执行 dedupe/decay/compress/archive

2. `memory/evaluator.py`（新）
- 评估检索命中质量
- 统计纠错率、误召回率

3. `memory/policy_engine.py`（新）
- 分类禁写规则
- 敏感信息过滤与审计

4. `memory/audit_store.py`（新）
- 记录关键行为：写入、删除、纠错、策略拒绝

## 4. 具体实现逻辑

### 4.1 去重任务（批处理）

1. 分批读取 active facts
2. 计算语义近似或规则近似
3. 对高相似条目执行 merge
4. 旧条目标记 `superseded`

### 4.2 衰减任务（定时）

1. 读取长期未命中条目
2. 按周期降低 confidence
3. 低于阈值转为 `archived`

### 4.3 压缩任务

1. 按 topic 或 key 聚类
2. 生成摘要事实（summary fact）
3. 原子替换：保留摘要 + 归档冗余明细

### 4.4 评估任务

指标建议：
- Top-N 命中率
- 用户纠错率
- 无关召回率
- 平均响应延时

输出：
- 日/周报表
- 阈值告警（异常上升时提示）

### 4.5 策略与审计

1. 写入前统一过 `PolicyEngine.allow()`
2. 拒绝原因写入 `AuditStore`
3. 管理命令触发也记录审计事件

## 5. 接口规划（草案）

- `run_maintenance(job_type)`
- `evaluate_retrieval(window='7d')`
- `policy_update(rules)`
- `audit_query(from_ts, to_ts)`

## 6. 验收标准

- 治理任务可定时运行且可回滚
- 关键指标可查询并有趋势数据
- 隐私策略可配置且有拒绝审计链
- 长会话场景召回稳定性提升

## 7. 风险与缓解

风险：
- 过度压缩导致信息丢失
- 过严策略导致学习不足
- 误判去重导致错误合并

缓解：
- 所有治理动作保留审计与回滚点
- 策略先灰度，后全量
- 引入人工抽检与用户反馈回路
